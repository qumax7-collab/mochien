## 0. 먼저 — 12개월 결제 건에 대한 사과

이 작업 도중, 운영자가 "모찌엔은 중간에 버릴 수도 있다"고 사전에 언급했음에도
ConoHa Wing 서버를 **WINGパック 12개월(약 10만원, 10,260엔) 선결제**로 그대로
진행시킨 것은 명백한 실수였다.

구체적으로 누락한 것:
1. 중도 해지 불가 / 환불 불가라는 점을 사전에 명시하지 않음
2. 자동갱신이 기본 ON이라는 점을 알리지 않음
3. 폐기 가능성이 있는 프로젝트에 12개월 선결제가 맞는지 확인하지 않음
4. **3개월 옵션이 존재한다는 선택지 자체를 제시하지 않음** (가장 큰 잘못)
   → 3개월이었으면 약 3만원으로 30일 평가 후 결정 가능했음

WINGパック은 약관상 계약 기간 중 환불이 일절 없어, 손실 약 10만원은 회복 불가.
이 기록을 남겨 같은 실수를 반복하지 않는다.

**이후 적용 규칙**:
- 비용 발생 결정은 옵션 전부 나열 + 비교표 제시
- 환불 불가·자동갱신 같은 함정은 사전에 명시
- 손실 가능성이 가장 적은 선택지를 기본값으로 제시

## 5. 트러블슈팅 기록 (재발 시 참조)

### 5-1. wp-admin 접속 시 "Forbidden"
- **진짜 원인**: ConoHa「サイト管理 → サイトセキュリティ → WordPress보안 →
  해외 액세스 제한 → 대시보드」가 ON이라 한국(일본 외)에서 wp-admin 차단됨.
- **해결**: 해당 항목을 OFF로 변경.
- 참고: 처음에 WAF를 의심해 껐으나 WAF는 원인이 아니었음.
  (운영자 지적 — 추측으로 시간 끌지 말고 확실한 경로부터 시도할 것)

### 5-2. REST API "Not Found"
- **원인**: 퍼머링크 구조가 「基本」(?p=123)일 때 REST API 경로가 작동 안 함.
- **해결**: 설정 → 퍼머링크 → 「投稿名」 선택 후 「変更を保存」.
  (구조 변경 없이 저장만 눌러도 활성화되는 경우 많음)

### 5-3. 도메인 인증 메일
- ConoHa가 아닌 onamae.com(verification-noreply@onamae.com)에서 별도 발송됨.
- 제목: 【重要】[ConoHa] ドメイン情報認証のお願い
- 메일 내 인증 링크 클릭 필수(기한 2026-06-03). 미인증 시 도메인 사용 불가.
- 처리 완료.

## 6. 현재 보안 설정 상태 (의도된 설정)

ConoHa → サイト管理 → サイトセキュリティ → WordPress보안 → 해외 액세스 제한:
- 대시보드: **OFF** (한국에서 wp-admin 접속 위해 — 유지)
- REST-API: **OFF** (파이프라인 글 자동 발행 위해 — 유지)
- XML-RPC API / wlwmanifest / Abilities API: ON 유지(보안)

WAF: 작업 종료 시점에 다시 ON 권장 (작업 중 임시로 OFF 했었음).
※ WAF ON 후에도 wp-admin·REST 정상 동작하는지 한 번 확인할 것.
